Personal Project · Mobile · React Native

O'Cañamos?

2025 GitHub ↗
Overview

Single-device party game for Android groups. One phone is passed hand-to-hand between players — each turn, a slot machine reveals one of 12 minigames with completely different mechanics, animations, and rules. Built as a personal challenge to push React Native's animation system and manage complex, branching game state without any third-party state management or audio libraries.

Tech Stack
React Native Expo SDK 54 React Navigation v7 Animated API useReducer + Context API Vibration API expo-screen-orientation EAS Build
Technical Challenges

12 different game states, one state machine

The core challenge: each minigame has its own internal state, win/lose conditions, and transitions — but they all live inside a single global useReducer that tracks turns, player order, the special card inventory, mirror effects, and the running counter. Every screen reads from this shared context and dispatches typed actions — no prop drilling, no external state library.

Animated API — no libraries
The slot machine reveal, dice roll, splash screen, and chaos button are all built with React Native's built-in Animated API. No Reanimated, no Lottie, no external dependencies — just Animated.Value, timing(), and spring().
Single useReducer state machine
All game state — players, turns, scores, card inventories, mirror effects — is managed through a single typed useReducer + Context tree. Each minigame dispatches typed actions; the reducer handles all valid transitions.
Haptic feedback, no audio
Every interaction has physical feedback through the Vibration API with custom vibration patterns per event type. Zero audio files — no permission requests, no file size overhead.
Special card system
Cards earned in the Auction minigame are stored in each player's inventory and can be played from the end-of-round screen. Each card type has a different effect on game state — shields, doubles, mirrors, wildcards — all handled as state transitions.
EAS Build → Android APK
The project uses EAS (Expo Application Services) with a preview profile that generates an unsigned APK for direct sideloading, and a production profile with auto-increment versioning for Play Store.
portrait locked + screen orientation
Uses expo-screen-orientation to lock the app in portrait mode globally — important for the hand-to-hand UX where the device changes orientation as it's passed between players.
Minigames

12 distinct mechanics

01
🎲 Todis
Animated dice roll. Each face maps to a different outcome. Face 6 triggers the mirror effect.
02
🃏 Cards
Challenge or question deck. Pass or accumulate a counter. 48 unique cards.
03
🧠 Trivia
3 attempts to answer at least 1 of 2 questions (easy/hard). 47 questions across categories.
04
🙅 Never Have I Ever
3 statements per round. Players self-identify. 49 unique prompts.
05
🤔 Who's Most Likely
Group vote mechanic. 3 questions, group nominates a player each time. 48 questions.
06
💬 Truth or Dare
Truth with 3 attempt limit, or skip and pay the counter. 40 unique truths.
07
🔫 Russian Roulette
6-chamber animation. Device passed hand-to-hand. The "bullet" chamber triggers a full glass.
08
💀 Elimination Roulette
Elimination mechanic. Players fall in sequence. Last standing loses.
09
🏷️ Auction
30-second timer auction. Highest bidder wins a special card. Entry point for the card inventory system.
10
🔴 Chaos Button
Animated chaos button. Random outcome — positive or negative — revealed on press.
11
🎓 Trivia Challenge
General knowledge questions with per-question penalty scaling.
12
🤥 Liar's Hour
Deduction round. One player bluffs, group detects. Wrong reads accumulate counter.
Special Cards

Inventory system

Cards earned at the Auction are stored per player and can be activated from the end-of-round inventory screen. Each card dispatches a specific state action that modifies the next round's behavior.

ShieldBlocks the next counter hit for the holder.
Double PenaltyDoubles the next counter hit received by target.
ImmunityFull immunity for the holder's next round.
RematchReplays the previous minigame.
WildcardHolder assigns their counter hit to any other player.
Reverse MirrorTransfers the mirror effect to another player.
